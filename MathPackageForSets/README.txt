#Краткая документация по Линейно/Дискретным алгоритмам

Программа представляет из себя набор классов для решения линейных и целочисленных задач.
1) Симплекс метод полностью работает, за исключением случая Креко, когда возникает вечный цикл из-за двух одинаковых чисел,
которые претендуют на звание разрешающих.
2) Первый метод Гомори. Работает не со всеми случаями.
3) Метод Ленд и Дойг. Не всегда корректно освобождает память.

Основные инструкции по использованию.
Пространство имен data предоставляет функцию fileReader(inputdata, file, PAMAMS).
inputdata - структуры инпут даты, которая содержит разобранное выражение.
file - входной файл.
PARAMS - режим ввода. По умолчанию стандартный.
Форматы ввода: 
0 1 0 -9 -> max (min) - DEFAULT
0x1 + x2 + 0x3 - 9x4 -> max (min) - WITHPARAMS

Линейные ограничений вводятся аналогично, только вместо стрелочки знаки отношений, а вместо цели задачи, - параметры свободного вектора.
Нули должны указываться явно.

Для работы программы необходимо создать экземпляр соответствующего класса (Simplex, Gomori, BorderMethod)
Запустить generate_plan(inputdata, MODE).
inputdata - структуры инпут даты, которая содержит разобранное выражение (полностью считанное и проверенное)
MODE - режим вывода (по умолчанию никакой, с onFile - в файл table.txt)

После подготовки остается вызвать метод .run() от экземпляра класса.
